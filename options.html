How to Define Generation Rules

The extension will automatically generate usernames and passwords by extracting variables from the URL and concatenating them with custom fields based on the generation rules.

By flexibly combining custom fields and URL variables, you can generate unique usernames and passwords for each website that are also easy to remember.

Example:

• Website: www.google.com

• Generated Username: user_goog_LE_123

• Generated Password: pass_GOOG_leD_321!

As long as you can remember your own account password generation rules, you don't need a password book to remember usernames and passwords for different websites.

URL Variable Reference Format:

{{[variable][startIndex][endIndex][case]}}

  • Index: Starts from 0, negative numbers count from the end
  • Case: U(uppercase), L(lowercase)

Example Explanation:

{{[domain][0][2][U]}} → Extracts characters 0~2 from the main domain and converts the result to uppercase

Currently Available Variables:

domain : The main domain part of the current page, automatically extracted from the URL. Example: For URL "https://www.example.com/path", the domain variable is "example"
Note: Originally planned to add other variables such as website name, path, parameters, etc., but due to complexity and practicality considerations, they are temporarily not added

Usage Examples:

Assuming the current page URL is: "https://www.example.com/path/to/page.html"

Generation Rule:{{[domain][0][2][U]}}
Generation Result:EXA
Generation Rule:{{[domain][-2][-1][L]}}
Generation Result:le
Generation Rule:{{[domain][1][3][U]}}
Generation Result:XAM
Generation Rule:PASSWORD_{{[domain][1][3][U]}}
Generation Result:PASSWORD_XAM
Generation Rule:{{[domain][0][1][U]}}_PASSWORD_{{[domain][1][3][U]}}_%@!123
